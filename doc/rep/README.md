# Radiant Enhancement Proposals (REPs)

This directory contains Radiant Enhancement Proposals (REPs), which are design documents providing information to the Radiant community or describing new features for Radiant.

## REP Categories

- **Standard Track (REP-1 to REP-999)**: Protocol changes, consensus rules, network changes
- **Informational (REP-1001 to REP-1999)**: Design issues, guidelines, informational content  
- **Process (REP-2001 to REP-2999)**: Process changes, governance, REP process itself

## Current REPs

*No REPs have been formally accepted yet. This section will be updated as REPs are submitted and approved.*

## Submitting a REP

1. Read the [REP Process](../rep-process.md) document
2. Use the [REP Template](../rep-template.md) as a starting point
3. Discuss your idea on community channels first
4. Submit your REP as a pull request to this repository

## REP Status Legend

- **Draft**: Initial proposal, being discussed
- **Active**: Under consideration, implementation may begin
- **Final**: Accepted and implemented
- **Rejected**: Considered and rejected
- **Withdrawn**: Author withdrew the proposal
